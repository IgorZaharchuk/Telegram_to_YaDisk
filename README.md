# Telegram to YaDisk Backup

Автоматическое резервное копирование файлов из Telegram на Яндекс.Диск с поддержкой тем, сжатием медиа и управлением через Telegram бота.

## 🚀 ВОЗМОЖНОСТИ

- **MTProto**: скачивание файлов до 2 GB без лимита Telegram Bot API
- **Темы**: автоматическое создание папок по названиям тем, независимое отслеживание прогресса для каждой темы
- **Гибкий выбор**: можно выбрать всю тему целиком или отдельные файлы
- **Фильтрация и сортировка**: быстрый поиск нужных файлов в интерфейсе бота
- **Сжатие**: фото в JPEG и видео в H.265 для экономии места до 70%
- **Управление через бота**: полноценное меню с кнопками, просмотр очереди, статистики и логов
- **Динамические лимиты**: отображение активных файлов (до 10), истории (до 10) и очереди (до 20) с пропорциональным распределением
- **Умные повторные попытки**: экспоненциальная задержка при ошибках, неисправимые ошибки не повторяются (максимум 3 попытки)
- **Восстановление после краша**: автоматическое восстановление несогласованных задач при перезапуске
- **Квота диска**: проверка свободного места перед загрузкой
- **Адаптивные воркеры**: автоматическая регулировка количества воркеров в зависимости от загрузки CPU
- **Прогресс-бары**: реальное время для скачивания, сжатия и загрузки (для файлов > 10 МБ)
- **Graceful shutdown**: корректное завершение с остановкой всех процессов ffmpeg
- **Безопасность**: защищённые права доступа к .env и сессиям, транзакции в БД
- **Systemd**: поддержка автозапуска через сервис Linux

## 📦 ТРЕБОВАНИЯ

- Python 3.8+
- FFmpeg (для сжатия видео)
- cpulimit (опционально, для ограничения CPU при сжатии)
- libheif-examples (для конвертации HEIC)
- SQLite3
- Система Linux (рекомендуется) или macOS

🔧 РЕЖИМЫ УСТАНОВКИ

При запуске install.sh вам будет предложено выбрать один из трёх режимов:

Режим 1: Клонирование в ~/Telegram_to_YaDisk

· Полная установка с клонированием репозитория в домашнюю директорию
· В конце будет предложено настроить systemd сервис для автозапуска
· Рекомендуется для большинства пользователей

Режим 2: Установка в текущую папку

· Использует существующие файлы в текущей директории
· Требует наличия requirements.txt и .env.example

Режим 3: Настройка systemd сервиса

· Только настройка автозапуска для уже установленной версии
· Используется если бот уже установлен и настроен, но нужно добавить автозапуск

🔑 ПОЛУЧЕНИЕ API CREDENTIALS

Telegram API

1. Перейдите на my.telegram.org/apps
2. Войдите под своим номером телефона
3. Создайте приложение
4. Скопируйте API_ID и API_HASH

STRING_SESSION

В папке util/ находится скрипт для создания сессии:

```bash
cd Telegram_to_YaDisk
source venv/bin/activate
python util/pyro_session_maker.py
```

Скрипт запросит:

· API_ID — из предыдущего шага
· API_HASH — из предыдущего шага
· Номер телефона в формате +7XXXXXXXXXX

После ввода кода подтверждения вы получите длинную строку — это ваша STRING_SESSION.

Яндекс.Диск Token

1. Перейдите на oauth.yandex.ru/client/new
2. Создайте новое приложение
3. Выберите права: «Яндекс.Диск. Доступ к папке приложения и чтение/запись на Диске»
4. Получите токен через OAuth flow

ID чатов и пользователей

1. Напишите боту @userinfobot
2. Перешлите ему любое сообщение из нужного чата
3. Бот покажет ID чата и ID отправителя

Управление чатами

Бот поддерживает добавление чатов по ID:

1. Настройки → Управление чатами → Добавить чат
2. Отправьте ID в формате -1001234567890
3. ID можно получить у @userinfobot

Выбор тем

В меню «Управление чатами» → выберите чат → «Управление файлами»:

· Выбрать тему — выбирает всю тему целиком (новые файлы будут добавляться автоматически)
· Открыть файлы — выбор отдельных файлов внутри темы

Выбор файлов

В меню управления файлами доступны:

· Фильтры: Все, Фото, Видео, Аудио, Документы, Архивы
· Сортировка: по дате, имени, размеру
· Выбор: выбрать всё, снять всё, выбрать новые, выбрать нескачанные
· Сброс ошибок: повторная попытка для файлов с ошибками

Админ-панель

В меню «Настройки» → «Админ-панель»:

· Завершить main.py — принудительная остановка бэкапа
· Очистить очередь — удалить все задачи из очереди
· Очистить temp — удалить временные файлы
· Сбросить статистику — полный сброс всей статистики и списка чатов

🔄 ЛОГИКА РАБОТЫ

Сканирование

· Полное сканирование (--full-scan): получает все сообщения чата, сохраняет в БД со статусом STATE_UNLOADED
· Инкрементальное сканирование: находит только новые файлы, сохраняет со статусом STATE_NEW

Очередь обработки

1. Check — проверка существования файла на Яндекс.Диске
2. Download — скачивание из Telegram
3. Compress — сжатие (фото в JPEG, видео в H.265)
4. Upload — загрузка на Яндекс.Диск

Восстановление после сбоев

· При перезапуске система автоматически находит несогласованные задачи
· Задачи в статусах pending_download, pending_compress откатываются на предыдущий этап
· Задачи в pending_upload сохраняют статус (файл уже готов к загрузке)
· После 3 неудачных попыток файл помечается как FAILED

Окна работы

· Настраиваются в боте: Настройки → Окна работы
· Можно задать несколько временных интервалов (например, 02:00–06:00)
· При выходе из окна бэкап получает 5 минут на завершение текущего файла, затем останавливается

🔧 УСТРАНЕНИЕ НЕПОЛАДОК

Ошибка авторизации Telegram

· Проверьте API_ID и API_HASH
· Получите новую STRING_SESSION через util/pyro_session_maker.py

Ошибка авторизации Яндекс.Диска

· Проверьте YA_DISK_TOKEN
· Получите новый токен в OAuth Яндекс

Бот не запускается

```bash
# Проверьте логи
tail -f logs/bot.log

# Проверьте наличие .env
ls -la .env

# Проверьте права доступа
chmod 600 .env

# Активируйте виртуальное окружение
source venv/bin/activate
```

Файлы не загружаются

```bash
# Проверьте логи бэкапа
tail -f logs/backup.log

# Проверьте квоту Яндекс.Диска
curl -H "Authorization: OAuth $YA_DISK_TOKEN" https://cloud-api.yandex.net/v1/disk

# Проверьте статус очереди
sqlite3 backup.db "SELECT status, COUNT(*) FROM queue_items GROUP BY status;"
```

Процессы ffmpeg не завершаются

```bash
# Принудительно убить все процессы ffmpeg
pkill -9 -f ffmpeg
pkill -9 -f cpulimit

# Или через бота: Админ-панель → Завершить main.py
```

Бот не отвечает

```bash
# Проверьте статус сервиса
sudo systemctl status tg2ya-bot

# Проверьте логи
sudo journalctl -u tg2ya-bot -f

# Перезапустите бота
pkill -f telegram_bot.py
./run_bot.sh
```

🐧 НАСТРОЙКА SYSTEMD

Создание сервиса

```bash
# Скопировать шаблон
sudo cp util/telegram-bot.service /etc/systemd/system/tg2ya-bot.service

# Перезагрузить systemd
sudo systemctl daemon-reload

# Включить автозапуск
sudo systemctl enable tg2ya-bot.service

# Запустить
sudo systemctl start tg2ya-bot.service
```

Управление сервисом

```bash
# Статус
sudo systemctl status tg2ya-bot

# Остановить
sudo systemctl stop tg2ya-bot

# Перезапустить
sudo systemctl restart tg2ya-bot

# Логи
sudo journalctl -u tg2ya-bot -f
```
